Create `src/ledger/integrity.ts` with two verification functions:

**1. `verifyLedgerChain(entriesPath: string): LedgerChainResult`**
Chain-only verification (fast, no decryption required, no key ring needed). Adapted from `src/audit/integrity.ts` verifyChain pattern.

```typescript
export interface LedgerChainResult {
  valid: boolean;
  entries: number;
  brokenAt?: number;
  error?: string;
}
```

Implementation:
1. If file doesn't exist or is empty, return { valid: true, entries: 0 }
2. Read file, split by newlines, filter empty
3. For each line:
   a. Parse as JSON (if fails -> return { valid: false, brokenAt: i, error: 'Malformed JSON at line N' })
   b. Verify parsed.prev_hash matches expectedPrevHash (first entry must have null)
   c. Compute SHA-256 of the raw line string, set as next expectedPrevHash
4. Return { valid: true, entries: lineCount }

This is essentially the same pattern as `src/audit/integrity.ts` but operates on entries.jsonl.

**2. `verifyLedgerIntegrity(entriesPath: string, getKeyById: (keyId: string) => Buffer): LedgerIntegrityResult`**
Full verification (chain + signatures + decryption). Expensive but comprehensive.

```typescript
export interface LedgerIntegrityResult {
  valid: boolean;
  entries: number;
  brokenAt?: number;
  error?: string;
  errorType?: 'chain' | 'signature' | 'decryption' | 'schema' | 'json';
}
```

Implementation:
1. If file doesn't exist or is empty, return { valid: true, entries: 0 }
2. Read file, split by newlines, filter empty
3. For each line:
   a. Parse as JSON (json error if fails)
   b. Verify hash chain (chain error if fails)
   c. Call readEntry(line, getKeyById) which internally decrypts and verifies signature
   d. If readEntry throws SignatureVerificationError -> return { valid: false, brokenAt: i, errorType: 'signature', error: message }
   e. If readEntry throws CryptoError -> return { valid: false, brokenAt: i, errorType: 'decryption', error: message }
   f. If readEntry throws LedgerCorruptedError -> return { valid: false, brokenAt: i, errorType: 'schema', error: message }
4. Return { valid: true, entries: lineCount }

Create `test/unit/ledger-integrity.test.ts` with tests:

1. **Valid chain**: Write 3 entries. verifyLedgerChain returns { valid: true, entries: 3 }.
2. **Missing file**: verifyLedgerChain on nonexistent path returns { valid: true, entries: 0 }.
3. **Empty file**: verifyLedgerChain on empty file returns { valid: true, entries: 0 }.
4. **Tampered entry (chain break)**: Write 3 entries. Modify the second entry's content in the file. verifyLedgerChain returns { valid: false, brokenAt: 2, error: contains 'Chain broken' }.
5. **Inserted entry**: Write 2 entries. Insert a fake entry between them. verifyLedgerChain detects the break.
6. **Deleted entry**: Write 3 entries. Delete the second entry. verifyLedgerChain detects the break.
7. **Malformed JSON**: Write 2 entries. Append a malformed line. verifyLedgerChain detects { brokenAt: 2, error: 'Malformed JSON' }.
8. **Genesis with non-null prev_hash**: Write a valid entry but manually modify its prev_hash to a non-null value. verifyLedgerChain detects break at entry 0.
9. **Full integrity - valid**: Write 3 entries. verifyLedgerIntegrity returns { valid: true, entries: 3 }.
10. **Full integrity - signature tamper**: Write an entry. Tamper with the signature field in the raw file. verifyLedgerIntegrity returns { valid: false, errorType: 'signature' }.
11. **Full integrity - payload tamper**: Write an entry. Tamper with the encrypted_payload ciphertext. verifyLedgerIntegrity returns { valid: false, errorType: 'decryption' }.

Test helper: Reuse the same temp dir + KeyRing + LedgerWriter helper from the writer tests. For tamper tests, read the file, modify the specific line, write it back.

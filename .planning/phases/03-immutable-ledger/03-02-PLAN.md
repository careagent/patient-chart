Create `src/ledger/writer.ts`:

```typescript
export class LedgerWriter {
  private lastHash: string | null = null;

  constructor(private readonly entriesPath: string) {
    this.lastHash = this.recoverLastHash();
  }
```

The LedgerWriter class:
- Constructor takes `entriesPath` (path to entries.jsonl). Recovers lastHash from existing file using the same crash-recovery pattern as AuditWriter (walk backward from end, skip malformed trailing lines).
- `writeEntry(content: unknown, entryType: LedgerEntryType, author: EntryAuthor, getActiveKey: () => { keyId: string; key: Buffer }, signingKey: KeyObject, opts?: { amends?: string }): LedgerEntry`
  1. Generate id (UUIDv7), timestamp (ISO 8601 ms precision)
  2. Build metadata: { schema_version: '1', entry_type: entryType, author_type: author.type, author_id: author.id, amends: opts?.amends, payload_size: 0 }
  3. Serialize content: `const plaintext = JSON.stringify(content)`
  4. Set metadata.payload_size = Buffer.byteLength(plaintext, 'utf-8')
  5. Build SignableContent: { id, timestamp, entry_type: entryType, author, payload: plaintext, metadata }
  6. Canonicalize: `const canonical = canonicalize(signable)`
  7. Sign: `const signature = sign(canonical, signingKey)` (from src/encryption/ed25519.ts)
  8. Encrypt: `const { keyId, key } = getActiveKey()` then `const encryptedPayload = encrypt(Buffer.from(plaintext, 'utf-8'), key, keyId, aad)` where `aad = Buffer.from(JSON.stringify(metadata), 'utf-8')`
  9. Build entry (without prev_hash): { id, timestamp, entry_type: entryType, author, signature: signature.toString('base64'), encrypted_payload: encryptedPayload, metadata }
  10. Chain and append: inject prev_hash from lastHash, serialize to JSON, appendFileSync, update lastHash to SHA-256 of the raw line
  11. Return the complete LedgerEntry

- `getLastHash(): string | null` -- returns current lastHash
- `getEntryCount(): number` -- returns count of entries written (for future rotation trigger)

CRITICAL DESIGN DECISIONS:
- The writer accepts `getActiveKey` as a function (not KeyRing directly) for loose coupling, matching the KnowledgeStore pattern from Phase 9.
- The writer accepts `signingKey: KeyObject` (the Ed25519 private key) directly.
- metadata.payload_size is the byte length of the PLAINTEXT JSON string (before encryption), included in SignableContent before signing. The read path reconstructs the same value from the decrypted plaintext.
- EntryMetadata serialized as JSON is passed as AAD to AES-256-GCM encrypt. The decrypt path must reconstruct the same AAD from the entry's metadata field.
- Optional fields (amends, synced_entry) that are undefined should NOT appear in the serialized metadata for AAD. Use a consistent approach: only include defined fields.

Create `src/ledger/reader.ts`:

- `readEntry(line: string, getKeyById: (keyId: string) => Buffer): { entry: LedgerEntry; plaintext: unknown }`
  1. Parse line as JSON, validate against LedgerEntrySchema (use Value.Check)
  2. Reconstruct AAD from entry.metadata: `Buffer.from(JSON.stringify(entry.metadata), 'utf-8')`
  3. Look up key: `const key = getKeyById(entry.encrypted_payload.key_id)`
  4. Decrypt: `const decrypted = decrypt(entry.encrypted_payload, key, aad)`
  5. Reconstruct SignableContent from decrypted payload + entry fields
  6. Canonicalize the reconstructed SignableContent
  7. Import author's public key: `importEd25519PublicKey(Buffer.from(entry.author.public_key, 'base64'))`
  8. Verify signature: `verifySignature(canonical, signatureBuffer, authorPublicKey)`
  9. If signature invalid, throw SignatureVerificationError(entry.id)
  10. Return { entry, plaintext: JSON.parse(decrypted.toString('utf-8')) }

- `readAllEntries(entriesPath: string, getKeyById: (keyId: string) => Buffer): Array<{ entry: LedgerEntry; plaintext: unknown }>`
  1. Read the file, split by newlines, filter empty
  2. For each line, call readEntry
  3. Return array of results

CRITICAL: The AAD reconstruction in the read path MUST match the AAD used during write. Both paths use `JSON.stringify(entry.metadata)`. Since metadata is read from the JSONL line (which was serialized by JSON.stringify during write), the key order will match because JSON.parse preserves insertion order and the original was produced by JSON.stringify from the same object structure.

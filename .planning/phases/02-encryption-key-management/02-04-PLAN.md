**Implement KeyRing class** in `src/encryption/keyring.ts`:

The KeyRing class manages the lifecycle of encryption keys and the identity key pair. It is instantiated via static factory methods (not constructor).

```typescript
class KeyRing {
  // Factory methods
  static create(keyWrappingKey: Buffer): KeyRing
  static load(filePath: string, keyWrappingKey: Buffer): KeyRing

  // Persistence
  save(filePath: string): void

  // Key operations
  getActiveKeyId(): string
  getEncryptionKey(keyId: string): Buffer
  getActiveEncryptionKey(): { keyId: string; key: Buffer }
  rotate(): string  // returns new key ID

  // Identity
  getIdentityPublicKey(): KeyObject
  getIdentityPrivateKey(): KeyObject
  getIdentityPublicKeyDer(): Buffer

  // Cleanup
  destroy(): void  // zeros all key material in memory
}
```

**Implementation details:**

1. **`KeyRing.create(keyWrappingKey)`**: Generate a new 32-byte AES encryption key via `randomBytes(32)`. Encrypt it with the key-wrapping key using `aes.encrypt()`. Generate an Ed25519 identity key pair. Export the private key to DER and encrypt it with the key-wrapping key. Export the public key to DER (stored in clear). Assign a UUIDv7 key ID. Build the in-memory state: decrypted keys map, active key ID, identity KeyObjects.

2. **`KeyRing.load(filePath, keyWrappingKey)`**: Read `filePath` using `readFileSync('utf-8')`. Parse JSON. Validate against `KeyRingDataSchema` using `Value.Check()` from TypeBox. For each key record, decrypt `encrypted_key` using the key-wrapping key — if AES-GCM auth tag fails, throw `VaultAuthenticationError` (wrong passphrase). Decrypt the identity private key similarly. Import the identity public key from stored DER. If JSON parse fails or schema validation fails, throw `KeyRingCorruptedError`.

3. **`KeyRing.save(filePath)`**: Build `KeyRingData` JSON from in-memory state (re-encrypting all keys with the key-wrapping key for fresh IVs). Write to `${filePath}.tmp` using `writeFileSync`. Rename to `filePath` using `renameSync`. This write-then-rename pattern provides atomic writes per Pitfall 5 in RESEARCH.md.

4. **`rotate()`**: Generate a new 32-byte encryption key. Encrypt it with the key-wrapping key. Set `rotated_at` on the current active key to `new Date().toISOString()`. Add the new key record. Update `active_key_id`. Return the new key ID. Old keys remain in the ring for historical decryption.

5. **`getEncryptionKey(keyId)`**: Look up keyId in the decrypted keys map. If not found, throw `KeyNotFoundError`. Return the decrypted 32-byte Buffer.

6. **`destroy()`**: Call `buffer.fill(0)` on all decrypted key Buffers (master key, encryption keys, identity private key DER). Best-effort memory zeroing per research recommendations.

**Imports**: `aes.encrypt`/`aes.decrypt` from `./aes.js`, `generateEd25519KeyPair`/`exportEd25519PublicKey`/`exportEd25519PrivateKey`/`importEd25519PublicKey`/`importEd25519PrivateKey` from `./ed25519.js`, `generateUUIDv7` from `../util/uuidv7.js`, `KeyRingDataSchema`/`KeyRingData` from `../types/encryption.js`, `Value` from `@sinclair/typebox/value`, error classes from `./errors.js`, `readFileSync`/`writeFileSync`/`renameSync` from `node:fs`, `randomBytes` from `node:crypto`.

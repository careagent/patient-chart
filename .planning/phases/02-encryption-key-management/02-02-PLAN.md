<feature>
  <name>AES-256-GCM Encrypt/Decrypt</name>
  <files>src/encryption/aes.ts, test/unit/aes.test.ts</files>
  <behavior>
    TDD RED-GREEN-REFACTOR cycle for AES-256-GCM symmetric encryption.

    **Function signatures:**
    - `encrypt(plaintext: Buffer, key: Buffer, keyId: string, aad?: Buffer): EncryptedPayload`
    - `decrypt(payload: EncryptedPayload, key: Buffer, aad?: Buffer): Buffer`

    **Test cases (RED phase — write first, all must fail):**
    1. encrypt returns an EncryptedPayload with ciphertext, iv, auth_tag (all base64), and key_id
    2. encrypt + decrypt round-trip produces identical plaintext
    3. Each encrypt call produces a different IV (generate two, compare — must differ)
    4. decrypt with wrong key throws (auth tag mismatch)
    5. decrypt with tampered ciphertext throws
    6. decrypt with tampered auth_tag throws
    7. decrypt with tampered IV throws (produces wrong plaintext, auth tag fails)
    8. encrypt with AAD + decrypt with same AAD succeeds
    9. encrypt with AAD + decrypt with different AAD throws
    10. encrypt with AAD + decrypt without AAD throws
    11. encrypt without AAD + decrypt with AAD throws
    12. key must be exactly 32 bytes — encrypt with 16-byte key throws CryptoError
    13. encrypt handles empty plaintext (Buffer.alloc(0)) without error
    14. encrypt handles large plaintext (1MB) without error

    **Implementation (GREEN phase):**
    - `encrypt`: Generate 12-byte IV via `randomBytes(12)`. Create cipher with `createCipheriv('aes-256-gcm', key, iv)`. If aad provided, call `cipher.setAAD(aad)`. Concat update + final for ciphertext. Get auth tag via `cipher.getAuthTag()`. Return `{ ciphertext: base64, iv: base64, auth_tag: base64, key_id: keyId }`.
    - `decrypt`: Create decipher with `createDecipheriv('aes-256-gcm', key, iv)`. Set auth tag via `decipher.setAuthTag()`. If aad provided, call `decipher.setAAD(aad)`. Concat update + final. Wrap node:crypto errors in CryptoError with generic message (never leak key material in errors).
    - Validate key length is 32 bytes before operations — throw CryptoError if not.
    - Import EncryptedPayload type from `../types/encryption.js`. Import CryptoError from `./errors.js`.
  </behavior>
  <implementation>
    All crypto via node:crypto built-ins only: createCipheriv, createDecipheriv, randomBytes.
    IV is NEVER a parameter — always generated internally by encrypt. This is an architectural decision to prevent IV reuse.
    The encrypt function does NOT accept an IV parameter. The function signature enforces this by design.
    Wrap all node:crypto exceptions in CryptoError to provide a consistent error surface.
    Use Buffer for plaintext input/output (not strings) to avoid encoding ambiguity.
  </implementation>
</feature>

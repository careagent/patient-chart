```typescript
export class IndexManager {
  private byType = new Map<string, Set<string>>();
  private byAuthor = new Map<string, Set<string>>();
  private byAmends = new Map<string, Set<string>>();
  private byId = new Map<string, number>();
  private lastHash: string | null = null;
```

Methods:
- `addEntry(entry: LedgerEntry, lineNumber: number): void` -- Adds entry to byType (keyed by entry_type), byAuthor (keyed by metadata.author_id), byId (keyed by entry.id -> lineNumber). If metadata.amends is defined, adds to byAmends (keyed by amends target -> entry.id).
- `getByType(type: LedgerEntryType): string[]` -- Returns array of entry IDs for the given type, or empty array.
- `getByAuthor(authorId: string): string[]` -- Returns array of entry IDs for the given author, or empty array.
- `getByAmends(targetId: string): string[]` -- Returns array of amendment entry IDs for the target, or empty array.
- `getLineNumber(id: string): number | undefined` -- Returns line number for the entry ID.
- `getEntryCount(): number` -- Returns byId.size.
- `getLastHash(): string | null` -- Returns lastHash.
- `setLastHash(hash: string | null): void` -- Updates lastHash.

Persistence:
- `save(indexPath: string): void` -- Serializes to JSON (byType as Record<string, string[]>, byAuthor as Record<string, string[]>, byAmends as Record<string, string[]>, byId as Record<string, number>, entryCount: number, lastHash: string|null). Uses write-then-rename atomic pattern (matching KeyRing.save).
- `static load(indexPath: string): IndexManager` -- Reads and parses index.json, populates Maps. Throws LedgerCorruptedError if JSON is malformed. Returns empty index if file doesn't exist.
- `static rebuild(entriesPath: string): IndexManager` -- Reads entries.jsonl line by line, parses each as LedgerEntry, calls addEntry for each. Computes lastHash from the last raw line. Returns fully populated IndexManager. Handles empty/missing file gracefully.

**GREEN phase**: Implement the module.

Create `src/ledger/query.ts`:

```typescript
export interface LedgerQuery {
  entry_type?: LedgerEntryType | LedgerEntryType[];
  author_id?: string;
  date_range?: { from?: string; to?: string };
  amends?: string;
  limit?: number;
  offset?: number;
}
```

Export `queryEntries(entriesPath: string, index: IndexManager, query: LedgerQuery, getKeyById: (keyId: string) => Buffer): Array<{ entry: LedgerEntry; plaintext: unknown }>`

Implementation strategy:
1. **Candidate set reduction** using index:
   - If query.entry_type is specified, get candidate IDs from index.getByType() (union if array)
   - If query.author_id is specified, get candidate IDs from index.getByAuthor()
   - If query.amends is specified, get candidate IDs from index.getByAmends()
   - Intersect all candidate sets. If no index filters, use all IDs from index.
2. **Get line numbers** for candidate IDs from index.getLineNumber()
3. **Read the file** and extract only the lines at the candidate line numbers (not full file scan for indexed queries)
4. **Apply remaining filters** (date_range) on each candidate entry:
   - Compare entry.timestamp with date_range.from and date_range.to using string comparison (ISO 8601 strings sort chronologically)
5. **Decrypt and verify** each matching entry via readEntry()
6. **Apply limit/offset** on the results
7. **Return** array of { entry, plaintext }

IMPORTANT: For efficiency, read the file once as an array of lines, then access by line number. Do NOT read each line individually.
